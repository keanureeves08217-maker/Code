## STEP 2: SETUP LOCALLY (5 MINUTES)

### On Windows:
1. Extract ZIP file
2. Right-click folder → "Open PowerShell here"
3. Type: `npm install`
4. Create `.env.local` file (see below)
5. Type: `npm run dev`

### On Mac/Linux:
1. Extract ZIP file
2. Open Terminal
3. Navigate: `cd scam-recovery-app`
4. Type: `npm install`
5. Create `.env.local` file (see below)
6. Type: `npm run dev`

### Create `.env.local` File:

In the project root folder, create a new file named **`.env.local`** with this content:

```
GMAIL_USER=your-email@gmail.com
GMAIL_APP_PASSWORD=your-16-character-password
ADMIN_EMAIL=admin@example.com
```

Replace with your actual values. See "Getting Gmail Credentials" below.

### Test Locally:
- Open http://localhost:3000 in your browser
- You should see the recovery form
- Try submitting a test application
- Check that emails arrive

## STEP 3: GET GMAIL CREDENTIALS

### Generate Gmail App Password (Required):

1. **Go to Google Account:**
   - Visit [myaccount.google.com/security](https://myaccount.google.com/security)
   - Sign in if needed

2. **Enable 2-Step Verification:**
   - If not already enabled, set it up
   - Choose your authentication method

3. **Generate App Password:**
   - In Security page, find "App passwords"
   - Select "Mail" as the app
   - Select your device type (or "Windows Computer")
   - Google generates a 16-character password
   - Copy this password

4. **Add to .env.local:**
   - Paste the password as `GMAIL_APP_PASSWORD`
   - Example: `GMAIL_APP_PASSWORD=abcd efgh ijkl mnop`

### Important Notes:
- DO NOT use your main Gmail password
- Keep the 16-character password secure
- Never commit `.env.local` to GitHub
- Each deployment needs these variables configured

## STEP 4: DEPLOY TO VERCEL

### Option A: Vercel CLI (Fastest)

```bash
# Install Vercel CLI
npm install -g vercel

# Deploy to production
vercel
```

Follow the prompts:
- Log in to Vercel (or create free account)
- Confirm project name
- Select "Next.js" when asked
- Let it build and deploy

### Option B: GitHub + Vercel (Auto-Deploys)

1. **Create GitHub Repository:**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/yourusername/scam-recovery.git
   git push -u origin main
   ```

2. **Connect to Vercel:**
   - Go to vercel.com
   - Click "New Project"
   - Import your GitHub repository
   - Vercel auto-detects Next.js settings
   - Click "Deploy"

3. **Every push to GitHub auto-deploys!**

### Option C: Vercel Dashboard Manual

1. Go to vercel.com/dashboard
2. Click "New Project"
3. Upload project files
4. Configure environment variables
5. Deploy

## TROUBLESHOOTING

**Problem: "GMAIL_USER is undefined"**
→ Solution: Ensure `.env.local` file exists in project root

**Problem: Emails not sending**
→ Solution: Verify Gmail credentials and enable 2-Step Verification

**Problem: Port 3000 already in use**
→ Solution: Run `npm run dev -- -p 3001`

**Problem: Images not uploading**
→ Solution: Check file size (max 5MB) and format (PNG/JPG)

**Problem: Loading screen doesn't appear**
→ Solution: Clear browser cache or try incognito mode

**Problem: Deployment fails on Vercel**
→ Solution: Check build logs, ensure all env vars are set
