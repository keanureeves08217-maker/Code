## ENVIRONMENT VARIABLES SETUP

Create `.env.local` file in project root with:

```
GMAIL_USER=your-email@gmail.com
GMAIL_APP_PASSWORD=your-16-char-app-password
ADMIN_EMAIL=admin@example.com
```

### Getting Gmail App Password:
1. Go to myaccount.google.com/security
2. Enable 2-Step Verification
3. Select "App passwords"
4. Choose "Mail" and your device
5. Copy 16-character password

## WHAT HAPPENS WHEN FORM IS SUBMITTED

1. User fills out form (8 fields required)
2. User uploads proof of payment image
3. User clicks "Submit Application"
4. Form data validated
5. Compensation calculated (80% of total amount)
6. Full-screen loader appears (5 minutes)
7. After 5 minutes, result screen shows compensation
8. Admin receives detailed email with attachment
9. Applicant receives confirmation email

## FILE DESCRIPTIONS

### app/page.tsx
Main page that imports and renders the RecoveryForm component. Very simple - just the entry point.

### app/layout.tsx
Root layout that wraps entire app. Contains metadata for SEO, Google Analytics, and sets background color.

### app/globals.css
All global styles. Contains:
- CSS custom properties (variables) for theme
- Dark theme colors (blue primary, cyan accent)
- Tailwind CSS imports
- Typography settings

### app/api/submit-recovery/route.ts
API endpoint that:
- Receives form data + base64 image
- Validates all fields
- Calculates 80% compensation
- Sends admin email with attachment
- Sends applicant confirmation email
- Returns compensation amount

### components/recovery-form.tsx
Main form component with:
- Form state management
- All 8 input fields
- Image upload integration
- Loading/success state handling
- Error display
- Trust indicator cards at bottom

### components/image-upload.tsx
Image upload component with:
- Drag-and-drop support
- Click-to-browse
- Image preview
- File size validation (5MB max)
- Format validation (PNG/JPG only)
- Remove button

### components/processing-loader.tsx
5-minute loading screen with:
- SVG circular progress animation
- Real-time percentage display
- Countdown timer (5:00 to 0:00)
- Animated loading dots
- Professional messaging
- Prevents window close

### components/result-screen.tsx
Result display screen with:
- Success checkmark icon
- Large compensation amount display
- 5-step processing explanation
- Email confirmation notice
- Option to submit another application
- Download confirmation button

## CUSTOMIZATION GUIDE

### Change Compensation Percentage
In `app/api/submit-recovery/route.ts`, line 46:
```typescript
const compensationAmount = totalAmount * 0.8;  // Change 0.8 to desired percentage (0.5 for 50%, etc)
```

### Change Colors
In `app/globals.css`, update CSS custom properties:
```css
--primary: oklch(0.55 0.22 258.55);  /* Blue - change these values */
--accent: oklch(0.65 0.2 192.24);    /* Cyan */
```

### Change Loading Time
In `components/processing-loader.tsx`, line 9:
```typescript
const [timeRemaining, setTimeRemaining] = useState(300);  // 300 = 5 minutes, change to 60 for 1 minute, etc
```

### Change Form Fields
Edit `components/recovery-form.tsx` to add/remove fields from the form

## TROUBLESHOOTING

**Emails not sending:**
- Verify Gmail credentials
- Enable Less Secure Apps or use App Password
- Check ADMIN_EMAIL is valid
- Look for Gmail security warnings

**Port 3000 already in use:**
- Run: `npm run dev -- -p 3001`

**Module not found errors:**
- Delete node_modules and .next
- Run: `npm install`

**Environment variables undefined:**
- Restart dev server after creating .env.local
- Make sure variable names match exactly
